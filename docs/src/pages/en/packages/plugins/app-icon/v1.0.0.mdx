## Plugin App Icon

`@brandingbrand/code-plugin-app-icon` is a package that generates all the necessary app icon assets for both iOS and Android.

### Getting Started

<div class="flex w-full my-6">
  <CodeCopy args="yarn add -D @brandingbrand/code-plugin-app-icon" />
</div>

Add the above noted package and the plugin is available in the Flagship Code plugin ecosystem.

### Usage

After adding this plugin as a dev dependency it is necessary to add it to the Flagship Code plugins configuration via the `package.json`.

`package.json`

```
  "code": {
    "plugins": [
      "@brandingbrand/code-plugin-app-icon"
    ]
  },
```

Your Flagship Code environment should also be updated with the `@brandingbrand/code-plugin-app-icon` required props from the defined interface.

`.coderc/env/env.<env>.ts`

```
  codePluginAppIcon: {
    code: {
      appIconPath: 'assets/app-icon',
    },
  },
```

The interface in your `.coderc/env/env.<env>.ts` has the following props:

<div class="my-4 overflow-scroll">
| attribute        | description                                                  |
| :--------        | :----------                                                  |
| code.appIconPath      | path to your app icon assets relative to .coderc directory |
</div>

The assets must follow a strict naming scheme.

<div class="my-4 overflow-scroll">
| platform         | filename                            | 
| :--------        | :----------                         |
| ios              | ios-universal.png                   |
| android legacy   | android-legacy.png                  |
| android adaptive | android-adaptive-foreground.png     |
| verbose adaptive | android-adaptive-background.png     |
</div>
