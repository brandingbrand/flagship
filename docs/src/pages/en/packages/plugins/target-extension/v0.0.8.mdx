## Plugin Target Extension

`@brandingbrand/kernel-plugin-target-extension` is a package that configures all the necessary native code for a native splash screen.

### Getting Started

<div class="flex w-full my-6">
  <CodeCopy args="yarn add -D @brandingbrand/kernel-plugin-target-extension" />
</div>

Add the above noted package and the plugin is avaialble in the Kernel plugin ecosystem.

### Usage

After adding this plugin as a dev dependency it is necessary to add it to the kernel plugins configuration via the `package.json`.

`package.json`

```
  "kernel": {
    "plugins": [
      "@brandingbrand/kernel-plugin-target-extension"
    ]
  },
```

Your Kernel environment should also be updated with the `@brandingbrand/kernel-plugin-target-extension` required props from the defined interface.

`.kernelrc/env/env.<env>.ts`

```
  kernelPluginTargetExtension: {
    kernel: [
      {
        path: "assets/extensions/NotificationService",
        bundleId: "com.client.myapp.notification",
        provisioningProfileName:
          "My App Notification Provisioning Profile",
      },
    ],
  },
```

The interface in your `.kernelrc/env/env.<env>.ts` has the following props:

<div class="my-4 overflow-scroll">
| attribute                             | description                         |
| :--------                             | :----------                         |
| kernel.ios[i].path                    | path to extension assets            |
| kernel.ios[i].bundleId                | extension bundle identifier         |
| kernel.ios[i].provisioningProfileName | extension provisioning profile name |
</div>
